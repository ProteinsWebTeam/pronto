Errors are represented using labels, such as:

Spelling
“domian ”

The first part (Spelling) is the type of error; the second part (domian) identifies the word, character, etc. that is incorrect, between double quotes. If the error occurs more than once, the number of occurences is also displayed (e.g. 3×“domian ”).

The possible errors are:

Abbreviation
The abstract or the entry name contains one or more invalid abbreviations (e.g. N-terminus instead of N-terminal, whitespace before kDa).
Accession
The entry name contains an InterPro or member database accession.
Character
The abstract contains one or more non-ASCII characters.
Citation
The abstract contains or more invalid references (e.g. PMID in clear text, et el.), etc.)
Double quotes
The entry name contains double quotes.
Empty block
The abstract has an paragraph.
Gene symbol
The entry name contains a potential gene symbol.
Illegal term
The entry name contains one or more illegal term (e.g. proteins)
Invalid signature
An homologous superfamily entry contains one or more signatures that are not from CATH-Gene3D or SUPERFAMILY.
Link
The abstract contains one or more invalid links (e.g. the resource does not exist any more).
Lower case
The entry name starts with a lowe case character.
Name clash
Two entries have the same name, or names differing only by non-alphanumeric characters (e.g. P67PHOX and P67-PHOX).
No signature
The entry is checked, but does not contain any signature.
Punctuation
The abstract contains one or more punctuation errors (e.g. whitespace before full stop).
Spelling
The abstract or the entry name contains a common mispelling (e.g. enrty instead of entry).
Substitution
The abstract contains one or more invalid substitutions, most likely done by the database (e.g. apostrophe converted to comma).
Too short
The abstract is empty, or too short (less than 25 characters).
Unchecked parent/child
The entry is checked, but has a parent or a child which is not.
Underscore
The entry name contains an underscore instead of an hyphen.
